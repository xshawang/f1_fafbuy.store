**簡要描述：** 

- 用戶支付發起接口

**請求URL：** 
- ` /pay `
  
**請求方式：**
- POST 

**參數：** 

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----                           |
|商戶ID|uid|string(5)|商戶ID號|
|币别|currencyID|int|参数值见「币别」|
|訂單號|orderid|string(32)|商戶平台生成的訂單號，唯一|
|支付類型|channel|int|參數值見「支付類型」|
|異步通知地址|notify_url|string(100)|http/https開頭字符串，支付完成後，通過POST請求的形式將支付結果作為參數通知到商戶系統|
|同步返回地址|return_url|string(100)|http/https開頭字符串，支付完成後，通過GET請求的形式跳轉回商戶頁面|
|金額|amount|float|商家提交金額，接受小數點後兩位|
|會員實名編號|user_id|string(10)|商家會員『唯一』編號，建議使用手機門號以利比對|
|會員實名|user_name|string(20)|商家會員實名|
|會員客端IP|userip|string(40)|商家會員ip地址，請確保正確|
|時間戳|timestamp|int|GMT時間戳|
|自定義|custom|string(100)|{“phone”:”09123456789”,”email”:”test@gmail.com”,”postcode”:”100”,”address”:”台北市中正區”,”goods”:”商品明細”}|
|數據簽名|sign|string(32)|32位大寫MD5簽名值  |

 **返回示例**

``` 
  {
    "status": 10000,
    "result": {
        "transactionid":"123",
        "payurl":"https://www",
		"points":"1000"
    }.`
    "sign":"xxx"
  }
```

 **返回參數中答复資料說明** 

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----                           |
|狀態代碼|status |int   |錯誤代碼資訊請看後台  |
|答复資料|result |object   |當狀態碼為「成功」時返回JSON字串  |
|數據簽名|sign |string   |32位大寫MD5簽名值  |

 **返回參數中JSON字串說明** 

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----    
|金流交易單號|transactionid |long   |金流平台生成的訂單號，唯一  |
|收銀台連結|payurl |string   |直接打開此鏈接進行支付  |
|金額|points|float|此次交易金額|
