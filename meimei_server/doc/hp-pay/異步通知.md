	3. 只有交易支付完成後，才會發送該通知。
	4. 請不要對該地址進行過濾攔截，cookies、session等在此頁面會失效，即無法獲取這些數據。
	5. 該方式的調試與運行必須在服務器上，即互聯網能訪問。
	6. 商戶系統接收到異步通知以後，必須通過驗簽（驗證通知中的sign參數）來確保支付通知是由我方發送的。
	7. 程序執行完後，請務必返回“success”(不含雙引號，大小寫敏感)，否則我方會重複發送異步通知三次。

### 小提醒
1. 組成 sign 時，只針對 status 與 result 兩個參數（都是字串）做處置，請勿將 sign 一起加入加密
2. 切勿將 result 的 JSON 資料解開成物件後再行加密
3. 狀態代碼 即是代表此筆交易訂單的交易狀態，請以此為交易成功與否的依據
4. 回調狀態，除去10000，其餘皆代表交易失敗
