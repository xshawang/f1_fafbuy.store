**簡要描述：** 

- 商戶依照傳入參數查詢訂單

**請求URL：** 
- ` /orderquery `
  
**請求方式：**
- POST 

**參數：** 

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----                           |
|商戶ID|uid|string(5)|商戶ID號|
|時間戳|timestamp|int|GMT時間戳|
|數據簽名|sign|string(32)|32位大寫MD5簽名值  |

**特定搜尋參數：**

依據使用方式不同，請自行帶入以下參數
- 使用時，請加入上方參數序列中。
- 若不需使用，請勿帶入以下參數。

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----                           |
|訂單號|orderid|string(32)|商戶平台生成的訂單號|
|币别|currencyID|int|参数值见「币别」|
|頁碼|page|int|每次返回數據最高依照行數而定，超過依照頁碼切換|
|行數|row|int|每次返回數據最高行數，允許1～1000|
|起始時間|start|string(19)|查詢區間的起始時間|
|截止時間|end|string(19)|查詢區間的截止時間|
|時間條件|date_type|int|依照不同時間類型搜尋，0：依照創單時間、1：依照修改時間|


 **返回示例**

``` 
{
    "status": 10000,
    "result": {
        "totalCount": "14",
        "page": "1",
        "row": "1",
        "count": 1,
        "data": {
            "0": {
                "transactionid": "1637571701769611008",
                "orderid": "992357165692361129645",
				"currencyID":"30",
                "channel": "910",
                "amount": "50000.00",
                "real_amount": "49000.00",
                "status": "1",
                "bdate": "2021-11-22 17:01:32",
                "cdate": "2021-11-22 17:15:10"
            }
        }
    },
    "sign": "********"
}
```

 **返回參數說明** 

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----                           |
|狀態代碼|status |int   |錯誤代碼資訊請看後台  |
|答复資料|result |object   |當狀態碼為「成功」時返回JSON字串  |
|數據簽名|sign |string   |32位大寫MD5簽名值  |

 **返回參數中答复資料說明** 

|參數名|參數|類型|說明|
|:-----  |:-----|:-----|-----                           |
|金流交易單號|transactionid |int   |金流平台生成的訂單號，唯一  |
|訂單號|orderid |string   |商戶平台生成的訂單號，唯一  |
|币别|currencyID|int|参数值见「币别」|
|支付類型|channel|int|參數值見末尾頁「支付類型」與「代付類型」|
|金額|amount |float    |商家提交金額，範圍小數點後兩位 |
|實際金額|real_amount |float   |實際處理的商戶金額，範圍小數點後兩位  |
|狀態|status |int   |0：未處理、1：交易成功、2：處理中、3：交易失敗、4：操作失敗、5：提單失敗  |
|創立時間|bdate |string   |交易單創立時間  |
|修改時間|cdate |string   |交易單最後修改時間  |

 **備註** 
- 在只帶入 uid、timestamp 、sign 的情況下，預設搜尋當這個小時的第一頁、20筆資料
- 若有帶入orderid（非空字串的情況下） 將會無視時間、頁碼、行數的參數（只需帶入uid、timestamp 、sign、orderid），只搜尋特定交易單
- 所有與時間有關的格式皆為：（Y-m-d h:i:s）2020-01-01 00:00:00
- 返回參數中的答复資料是以有序物件組成
- 系统限制每 5 秒可查询一次
